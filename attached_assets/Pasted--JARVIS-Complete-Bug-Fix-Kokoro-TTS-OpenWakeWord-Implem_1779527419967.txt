## PART 5 — AUDIO STATE MACHINE: STUCK IN RESPONDING + NO IDLE RETURN

**File:** `src/lib/jarvis.ts`

**Problem 1:** After speaking finishes, state gets stuck in `speaking`/`responding` forever. Browser TTS `onend` event is unreliable — Chrome has a known bug where speechSynthesis pauses after ~15 seconds and `onend` never fires.

**Fix:** Replace the entire `TTSQueue.playNext()` method:

```typescript
private playNext(): void {
  if (this.queue.length === 0) {
    this.isPlaying = false
    transitionAudioState('idle')
    return
  }

  const sentence = this.queue.shift()!
  const utterance = new SpeechSynthesisUtterance(sentence)
  utterance.rate = 1.0
  utterance.pitch = 1.0

  // Estimated duration + buffer for timeout fallback
  const estimatedMs = Math.max((sentence.length / 12) * 1000, 3000) + 4000

  let timeoutId: ReturnType<typeof setTimeout> | null = setTimeout(() => {
    window.speechSynthesis.cancel()
    this.queue = []
    this.isPlaying = false
    transitionAudioState('idle')
  }, estimatedMs)

  const cleanup = () => {
    if (timeoutId) {
      clearTimeout(timeoutId)
      timeoutId = null
    }
  }

  utterance.onend = () => {
    cleanup()
    this.playNext()
  }

  utterance.onerror = () => {
    cleanup()
    this.isPlaying = false
    transitionAudioState('idle')
  }

  window.speechSynthesis.speak(utterance)
}
```

**Problem 2:** Chrome speechSynthesis pauses mid-speech after ~15s.

## PART 6 — INTERRUPT WORDS NOT WORKING

**File:** `src/lib/jarvis.ts`

**Problem:** "stop", "wait" etc don't interrupt Jarvis while speaking because no SpeechRecognition instance is listening during the `speaking` state.

**Fix:** Add a dedicated interrupt listener that activates whenever state transitions to `speaking`:

```typescript
let _interruptRecognition: SpeechRecognition | null = null

function startInterruptListener(): void {
  if (!('SpeechRecognition' in window || 'webkitSpeechRecognition' in window)) return
  
  try {
    const SR = (window as any).SpeechRecognition || (window as any).webkitSpeechRecognition
    _interruptRecognition = new SR()
    _interruptRecognition!.continuous = false
    _interruptRecognition!.interimResults = false
    _interruptRecognition!.lang = 'en-US'
    _interruptRecognition!.maxAlternatives = 1

    _interruptRecognition!.onresult = (event: SpeechRecognitionEvent) => {
      const transcript = event.results[0][0].transcript.toLowerCase().trim()
      const INTERRUPT_WORDS = ['stop', 'wait', 'hold on', 'shut up', 'pause', 'enough', 'quiet']
      
      if (INTERRUPT_WORDS.some(word => transcript.includes(word))) {
        ttsQueue.interrupt()
        stopInterruptListener()
        transitionAudioState('interrupted')
        setTimeout(() => {
          if (_state.audioState === 'interrupted') {
            transitionAudioState('idle')
          }
        }, 800)
      }
    }

    _interruptRecognition!.onerror = () => {
      // silent fail — interrupt listener is best-effort
    }

    _interruptRecognition!.onend = () => {
      // restart if still speaking
      if (_state.audioState === 'speaking') {
        try { _interruptRecognition?.start() } catch (e) {}
      }
    }

    _interruptRecognition!.start()
  } catch (e) {
    console.warn('[JARVIS] Interrupt listener failed to start:', e)
  }
}

function stopInterruptListener(): void {
  if (_interruptRecognition) {
    try { _interruptRecognition.stop() } catch (e) {}
    _interruptRecognition = null
  }
}
```

In `transitionAudioState()`:
- When transitioning TO `speaking` → call `startInterruptListener()`
- When transitioning AWAY from `speaking` → call `stopInterruptListener()`

```typescript
function cleanTextForSpeech(text: string): string {
  return text
    .replace(/\*\*(.*?)\*\*/g, '$1')
    .replace(/\*(.*?)\*/g, '$1')
    .replace(/#{1,6}\s+/g, '')
    .replace(/`{3}[\s\S]*?`{3}/g, 'code block omitted')
    .replace(/`([^`]+)`/g, '$1')
    .replace(/\[([^\]]+)\]\([^)]+\)/g, '$1')
    .replace(/^\s*[-*+]\s+/gm, '')
    .replace(/^\s*\d+\.\s+/gm, '')
    .replace(/\n{2,}/g, '. ')
    .replace(/\n/g, ' ')
    .replace(/\s{2,}/g, ' ')
    .trim()
}
```

**What Kokoro is:** An 82MB neural TTS model that runs entirely in the browser via WebAssembly. Voice quality is significantly better than browser TTS — sounds human, not robotic. No API key. No server. Free forever. Model downloads once and caches in IndexedDB permanently.

**Step 2 — Create Web Worker:**

Create `src/workers/kokoro-worker.js`:

```javascript
import { KokoroTTS } from 'kokoro-js'

let tts = null
let isLoading = false

self.onmessage = async (event) => {
  const { type, text, voice, id } = event.data

  if (type === 'load') {
    if (tts || isLoading) {
      self.postMessage({ type: 'load_status', ready: !!tts })
      return
    }
    
    isLoading = true
    self.postMessage({ type: 'loading', message: 'Downloading voice model (82MB, one-time)...' })
    
    try {
      tts = await KokoroTTS.from_pretrained(
        'onnx-community/Kokoro-82M-v1.0_fp16',
        { dtype: 'fp16' }
      )
      isLoading = false
      self.postMessage({ type: 'load_status', ready: true })
    } catch (err) {
      isLoading = false
      self.postMessage({ type: 'load_error', error: err.message })
    }
  }

  if (type === 'synthesize') {
    if (!tts) {
      self.postMessage({ type: 'synth_error', id, error: 'Model not loaded' })
      return
    }

    try {
      const audio = await tts.generate(text, {
        voice: voice || 'am_adam', // am_adam = American male, natural voice
      })
      
      // Convert to transferable ArrayBuffer
      const audioData = audio.audio
      const sampleRate = audio.sampling_rate
      
      self.postMessage(
        { type: 'synth_result', id, audioData, sampleRate },
        [audioData.buffer]
      )
    } catch (err) {
      self.postMessage({ type: 'synth_error', id, error: err.message })
    }
  }
}
```

**Step 3 — Kokoro Manager in `src/lib/jarvis.ts`:**

Add a `KokoroManager` class that manages the worker:

```typescript
class KokoroManager {
  private worker: Worker | null = null
  private isReady: boolean = false
  private isLoading: boolean = false
  private pendingCallbacks: Map<string, {
    resolve: (audioData: Float32Array, sampleRate: number) => void
    reject: (err: Error) => void
  }> = new Map()
  private onLoadingCallback: ((msg: string) => void) | null = null

  initialize(onLoading?: (msg: string) => void): void {
    if (this.worker || this.isLoading) return
    this.onLoadingCallback = onLoading || null
    this.isLoading = true

    try {
      this.worker = new Worker(
        new URL('../workers/kokoro-worker.js', import.meta.url),
        { type: 'module' }
      )

      this.worker.onmessage = (event) => {
        const { type, id, audioData, sampleRate, error, message, ready } = event.data

        if (type === 'loading') {
          this.onLoadingCallback?.(message)
        }

        if (type === 'load_status') {
          this.isReady = ready
          this.isLoading = false
        }

        if (type === 'load_error') {
          this.isReady = false
          this.isLoading = false
          console.warn('[Kokoro] Failed to load model:', error)
          // fallback to browser TTS handled in TTSQueue
        }

        if (type === 'synth_result') {
          const cb = this.pendingCallbacks.get(id)
          if (cb) {
            this.pendingCallbacks.delete(id)
            cb.resolve(new Float32Array(audioData), sampleRate)
          }
        }

        if (type === 'synth_error') {
          const cb = this.pendingCallbacks.get(id)
          if (cb) {
            this.pendingCallbacks.delete(id)
            cb.reject(new Error(error))
          }
        }
      }

      this.worker.onerror = (err) => {
        console.warn('[Kokoro] Worker error:', err)
        this.isReady = false
        this.isLoading = false
      }

      this.worker.postMessage({ type: 'load' })
    } catch (err) {
      console.warn('[Kokoro] Failed to initialize worker:', err)
      this.isLoading = false
    }
  }

  async synthesize(text: string, voice = 'am_adam'): Promise<{ audioData: Float32Array, sampleRate: number }> {
    if (!this.worker || !this.isReady) {
      throw new Error('Kokoro not ready')
    }

    const id = `synth_${Date.now()}_${Math.random()}`

    return new Promise((resolve, reject) => {
      this.pendingCallbacks.set(id, { resolve, reject })
      this.worker!.postMessage({ type: 'synthesize', text, voice, id })
    })
  }

  get ready(): boolean { return this.isReady }
  get loading(): boolean { return this.isLoading }

  destroy(): void {
    this.worker?.terminate()
    this.worker = null
    this.isReady = false
    this.pendingCallbacks.clear()
  }
}

export const kokoroManager = new KokoroManager()
```

**Step 4 — Update TTSQueue to use Kokoro:**

Replace `TTSQueue.playNext()` with a version that uses Kokoro when available, falls back to browser TTS when not:

```typescript
private async playNext(): Promise<void> {
  if (this.queue.length === 0) {
    this.isPlaying = false
    this.stopKeepAlive()
    transitionAudioState('idle')
    return
  }

  const sentence = this.queue.shift()!

  // Try Kokoro first
  if (kokoroManager.ready) {
    try {
      const { audioData, sampleRate } = await kokoroManager.synthesize(sentence)
      
      // Play via Web Audio API
      const audioContext = new AudioContext()
      const buffer = audioContext.createBuffer(1, audioData.length, sampleRate)
      buffer.getChannelData(0).set(audioData)
      
      const source = audioContext.createBufferSource()
      source.buffer = buffer
      source.connect(audioContext.destination)
      
      source.onended = () => {
        audioContext.close()
        this.playNext()
      }
      
      source.start(0)
      this.currentSource = source
      return
    } catch (err) {
      console.warn('[Kokoro] Synthesis failed, falling back to browser TTS:', err)
      // fall through to browser TTS
    }
  }

  // Browser TTS fallback
  const utterance = new SpeechSynthesisUtterance(sentence)
  utterance.rate = 1.0

  const estimatedMs = Math.max((sentence.length / 12) * 1000, 3000) + 4000
  let timeoutId: ReturnType<typeof setTimeout> | null = setTimeout(() => {
    window.speechSynthesis.cancel()
    this.queue = []
    this.isPlaying = false
    transitionAudioState('idle')
  }, estimatedMs)

  utterance.onend = () => {
    if (timeoutId) { clearTimeout(timeoutId); timeoutId = null }
    this.playNext()
  }
  
  utterance.onerror = () => {
    if (timeoutId) { clearTimeout(timeoutId); timeoutId = null }
    this.isPlaying = false
    transitionAudioState('idle')
  }

  window.speechSynthesis.speak(utterance)
}

interrupt(): void {
  this.queue = []
  
  // Stop Kokoro audio
  if (this.currentSource) {
    try { this.currentSource.stop() } catch (e) {}
    this.currentSource = null
  }
  
  // Stop browser TTS
  window.speechSynthesis.cancel()
  
  this.isPlaying = false
  this.stopKeepAlive()
  transitionAudioState('interrupted')
}
```

Add `private currentSource: AudioBufferSourceNode | null = null` to TTSQueue class properties.

## PART 12 — OPENWAKEWORD IMPLEMENTATION (REPLACES UNRELIABLE WEB SPEECH WAKE WORD)

**What OpenWakeWord is:** A completely free, open-source wake word detection library that runs in the browser via WebAssembly. Uses ONNX neural network models. Has `hey_jarvis` as a pre-trained built-in model. Zero API key. Zero signup. Zero cost. Significantly more reliable than the current Web Speech API wake word scanning loop.

**Step 1 — Install:**
```
npm install openwakeword-wasm-browser
```

**Step 2 — Replace wake word logic:**

Remove the existing Web Speech API passive listening loop that scans for "jarvis", "hey jarvis", "okay jarvis" using SpeechRecognition. This entire loop is what causes unreliable behavior — SpeechRecognition was never designed for always-on keyword spotting.

Replace with:

```typescript
import WakeWordEngine from 'openwakeword-wasm-browser'

let _wakeWordEngine: InstanceType<typeof WakeWordEngine> | null = null
let _wakeWordActive = false

async function startWakeWordDetection(): Promise<void> {
  // Only on desktop — mobile uses tap-to-talk
  if (isMobile()) return
  
  if (_wakeWordActive || _wakeWordEngine) return
  
  try {
    _wakeWordEngine = new WakeWordEngine({
      keywords: ['hey_jarvis'],
      detectionThreshold: 0.5,
      cooldownMs: 2000, // 2s cooldown between detections
    })

    await _wakeWordEngine.load()

    _wakeWordEngine.on('detect', ({ keyword, score }: { keyword: string, score: number }) => {
      console.log(`[WakeWord] Detected: ${keyword} (score: ${score.toFixed(2)})`)
      
      if (_state.audioState === 'idle' && _state.enabled) {
        transitionAudioState('listening')
        startCommandListening() // existing function that starts SpeechRecognition for command capture
      }
    })

    _wakeWordEngine.on('error', (err: Error) => {
      console.warn('[WakeWord] Error:', err)
    })

    await _wakeWordEngine.start()
    _wakeWordActive = true
    
    console.log('[JARVIS] Wake word detection active — say "Hey Jarvis"')
  } catch (err) {
    console.warn('[WakeWord] Failed to initialize:', err)
    // Graceful degradation — tap-to-talk still works
  }
}

async function stopWakeWordDetection(): Promise<void> {
  if (_wakeWordEngine) {
    try {
      await _wakeWordEngine.stop()
    } catch (e) {}
    _wakeWordEngine = null
  }
  _wakeWordActive = false
}
```

**Step 3 — Wire into enable/disable:**

When `jarvis.enable()` is called → call `startWakeWordDetection()`
When `jarvis.disable()` is called → call `stopWakeWordDetection()`
On JARVIS page unmount → call `stopWakeWordDetection()`

**Step 4 — Asset files:**

The `openwakeword-wasm-browser` package requires ONNX model files to be served from a public path. After installing the package, copy the model files to the Vite public directory:

In `vite.config.ts`, check if there's a way to copy assets. Otherwise manually copy the contents of `node_modules/openwakeword-wasm-browser/models/` to `public/openwakeword/models/`.

In the WakeWordEngine initialization, set:
```typescript
_wakeWordEngine = new WakeWordEngine({
  baseAssetUrl: '/openwakeword/models',
  keywords: ['hey_jarvis'],
  detectionThreshold: 0.5,
  cooldownMs: 2000,
})
```

If the package provides a way to copy assets automatically via a postinstall script or vite plugin, use that instead.

**Step 5 — Keep tap-to-talk working:**

The existing tap-to-talk functionality (clicking the orb) must continue working exactly as before. Wake word detection is additive — it adds a new way to trigger listening, it does not replace the orb tap. Both work simultaneously.
