PART 12 — OPENWAKEWORD IMPLEMENTATION (REPLACES UNRELIABLE WEB SPEECH WAKE WORD)
File: src/lib/jarvis.ts
What OpenWakeWord is: A completely free, open-source wake word detection library that runs in the browser via WebAssembly. Uses ONNX neural network models. Has hey_jarvis as a pre-trained built-in model. Zero API key. Zero signup. Zero cost. Significantly more reliable than the current Web Speech API wake word scanning loop.
Step 1 — Install:
npm install openwakeword-wasm-browser
Step 2 — Replace wake word logic:
Remove the existing Web Speech API passive listening loop that scans for "jarvis", "hey jarvis", "okay jarvis" using SpeechRecognition. Replace with:
typescriptimport WakeWordEngine from 'openwakeword-wasm-browser'

let _wakeWordEngine: InstanceType<typeof WakeWordEngine> | null = null
let _wakeWordActive = false

async function startWakeWordDetection(): Promise<void> {
  // Only on desktop — mobile uses tap-to-talk
  if (isMobile()) return
  if (_wakeWordActive || _wakeWordEngine) return

  try {
    _wakeWordEngine = new WakeWordEngine({
      baseAssetUrl: '/openwakeword/models',
      keywords: ['hey_jarvis'],
      detectionThreshold: 0.5,
      cooldownMs: 2000,
    })

    await _wakeWordEngine.load()

    _wakeWordEngine.on('detect', ({ keyword, score }: { keyword: string; score: number }) => {
      console.log(`[WakeWord] Detected: ${keyword} (score: ${score.toFixed(2)})`)
      if (_state.audioState === 'idle' && _state.enabled) {
        transitionAudioState('listening')
        startCommandListening()
      }
    })

    _wakeWordEngine.on('error', (err: Error) => {
      console.warn('[WakeWord] Error:', err)
    })

    await _wakeWordEngine.start()
    _wakeWordActive = true
    console.log('[JARVIS] Wake word detection active — say "Hey Jarvis"')
  } catch (err) {
    console.warn('[WakeWord] Failed to initialize:', err)
    // Graceful degradation — tap-to-talk still works
  }
}

async function stopWakeWordDetection(): Promise<void> {
  if (_wakeWordEngine) {
    try { await _wakeWordEngine.stop() } catch (e) {}
    _wakeWordEngine = null
  }
  _wakeWordActive = false
}
Step 3 — Wire into enable/disable:

When jarvis.enable() is called → call startWakeWordDetection()
When jarvis.disable() is called → call stopWakeWordDetection()
On JARVIS page unmount → call stopWakeWordDetection()

Step 4 — Asset files:
After installing the package, copy the model files to the Vite public directory. In vite.config.ts, add:
typescriptimport { viteStaticCopy } from 'vite-plugin-static-copy'

// Inside plugins array:
viteStaticCopy({
  targets: [
    {
      src: 'node_modules/openwakeword-wasm-browser/models/*',
      dest: 'openwakeword/models'
    }
  ]
})
If vite-plugin-static-copy is not already installed, add it. Otherwise manually copy the contents of node_modules/openwakeword-wasm-browser/models/ to public/openwakeword/models/.
Step 5 — Keep tap-to-talk working:
The existing tap-to-talk functionality (clicking the orb) must continue working exactly as before. Wake word detection is purely additive — it adds a new trigger path, it does not replace or interfere with the orb tap. Both work simultaneously on desktop. Mobile uses tap-to-talk exclusively.

CODE QUALITY STANDARDS
Every line of code written in this implementation must meet the following bar without exception:
Structure & Naming

All functions named with clear verb-noun intent: startWakeWordDetection, cleanTextForSpeech, transitionAudioState — never handleThing, doStuff, process
No magic numbers — every threshold, timeout value, and constant gets a named const at the top of its scope with a comment explaining why that value was chosen
No inline object literals repeated in multiple places — extract to a named constant

TypeScript

No any except at unavoidable browser API boundaries (window as any for SpeechRecognition) — and those must have an inline comment explaining why
Every function has explicit return type annotations
Every class property typed — no implicit any from missing annotations
No non-null assertions (!) without a preceding runtime guard that makes them safe

Error Handling

Every async function that touches external state (Supabase, Web Audio, Worker, SpeechRecognition) wrapped in try/catch
Every catch block either re-throws with context, logs with [MODULE] prefix, or silently swallows with an inline comment explaining why silence is correct
No empty catch blocks without a comment

React

No useEffect without a complete dependency array
No state updates after unmount — every effect that sets state checks a mounted ref or cleans up properly
No prop drilling past 2 levels — lift state or use the existing state module

General

No commented-out code in the final output
No console.log left in production paths — only console.warn for degraded states and console.error for genuine failures
Every file ends with a newline
Imports sorted: external packages first, then internal modules, then relative paths
