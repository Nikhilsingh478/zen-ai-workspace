# STEP 3 — Create src/lib/supabase-data.ts
This file replaces all localStorage I/O. Implement these functions:

fetchAllData(): Promise<AppStorage>
  - Query all 3 tables in parallel with Promise.all
  - Map rows to Website | Prompt types
  - Return full AppStorage shape

addWebsite(website: Website): Promise<void>
  - Insert into items table
  - Insert into desktop_layout at x:0, y:0

addPrompt(prompt: Prompt): Promise<void>
  - Insert into items table only

removeItem(id: string): Promise<void>
  - Delete from items
  - Delete from desktop_layout
  - Remove id from any folder's children array
  - Delete folder if it becomes empty

updateLayout(layout: DesktopLayoutEntry[]): Promise<void>
  - Upsert all entries into desktop_layout

createFolder(id: string, x: number, y: number): Promise<void>
  - Insert into desktop_folders
  - Insert into desktop_layout

updateFolder(id: string, updates: {name?: string, children?: string[]}): Promise<void>
  - Update desktop_folders row

deleteFolder(id: string, childEntries: DesktopLayoutEntry[]): Promise<void>
  - Delete from desktop_folders
  - Delete from desktop_layout
  - Upsert childEntries back into desktop_layout

# STEP 7 — Add Supabase realtime sync

After fetchAllData completes, subscribe to changes:

supabase
  .channel("db-changes")
  .on("postgres_changes", { event: "*", schema: "public", table: "items" }, 
    () => refetch())
  .on("postgres_changes", { event: "*", schema: "public", table: "desktop_layout" },
    () => refetch())
  .on("postgres_changes", { event: "*", schema: "public", table: "desktop_folders" },
    () => refetch())
  .subscribe()

Where refetch() calls fetchAllData() and updates state.
Unsubscribe on cleanup.
