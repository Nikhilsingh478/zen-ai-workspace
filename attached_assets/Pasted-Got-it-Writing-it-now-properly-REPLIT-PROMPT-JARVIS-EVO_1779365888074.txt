## PART 1 — DATABASE CHANGES (Supabase)

Run the following SQL. Do not touch any existing tables.

```sql
-- Conversation sessions table
create table if not exists jarvis_sessions (
  id uuid primary key default gen_random_uuid(),
  started_at timestamptz not null default now(),
  ended_at timestamptz,
  session_summary text,
  message_count integer not null default 0,
  tags text[] not null default '{}'
);

-- Individual messages within sessions
create table if not exists jarvis_messages (
  id uuid primary key default gen_random_uuid(),
  session_id uuid not null references jarvis_sessions(id) on delete cascade,
  role text not null check (role in ('user', 'assistant')),
  content text not null,
  message_type text not null default 'conversation',
  -- message_type options:
  -- 'conversation' = normal back and forth
  -- 'task_created' = jarvis created a task
  -- 'memory_saved' = jarvis saved a memory
  -- 'error' = error state
  metadata jsonb,
  created_at timestamptz not null default now()
);

-- Persistent memory / quick captures
create table if not exists jarvis_memory (
  id uuid primary key default gen_random_uuid(),
  content text not null,
  memory_type text not null default 'general',
  -- memory_type options:
  -- 'general' = anything
  -- 'preference' = user preference or habit
  -- 'commitment' = something user committed to do
  -- 'idea' = an idea captured
  -- 'fact' = factual thing about user's life
  source_session_id uuid references jarvis_sessions(id) on delete set null,
  recalled_count integer not null default 0,
  created_at timestamptz not null default now(),
  last_recalled_at timestamptz
);

-- Indexes for performance
create index if not exists idx_jarvis_messages_session 
  on jarvis_messages(session_id);
create index if not exists idx_jarvis_messages_created 
  on jarvis_messages(created_at desc);
create index if not exists idx_jarvis_memory_type 
  on jarvis_memory(memory_type);
create index if not exists idx_jarvis_memory_created 
  on jarvis_memory(created_at desc);
```

## PART 4 — SESSION MANAGEMENT

### File: `src/lib/jarvis.ts`

Add session management functions:

#### `startSession()` → `Promise<string>` (returns session ID)
- Creates a new row in `jarvis_sessions`
- Returns the session UUID
- Called when JARVIS page mounts or when user starts speaking for the first time after a period of inactivity (30+ minutes since last message)

#### `endSession(sessionId)` → `Promise<void>`
- Updates `jarvis_sessions` with `ended_at = now()`
- Sends a final request to Gemini asking it to write a 2-3 sentence summary of what was discussed in the session, focused on key decisions, tasks created, and things to remember
- Saves that summary to `jarvis_sessions.session_summary`
- Updates `message_count`
- Called when the user navigates away from the JARVIS page or closes the tab (use `beforeunload` event)

#### `saveMessage(sessionId, role, content, messageType, metadata?)` → `Promise<void>`
- Inserts a row into `jarvis_messages`
- Called after every user message and every JARVIS response
- `metadata` stores tool call info if relevant (e.g. `{ tool: "create_task", taskId: "..." }`)

#### `loadSessionMessages(sessionId)` → `Promise<Message[]>`
- Fetches all messages for a session ordered by `created_at`
- Used to restore conversation on page reload within the same session

#### `getRecentSessionSummaries(limit = 3)` → `Promise<string[]>`
- Returns the last N non-null session summaries
- Used when building the system prompt

#### `getRecentMemories(limit = 20)` → `Promise<Memory[]>`
- Returns most recent memories from `jarvis_memory`

## PART 7 — AUDIO STATE MACHINE

### File: `src/lib/jarvis.ts`

The existing voice state already has some states. Formalize it into a proper state machine.

Replace or extend the existing voice state with these exact states:

```typescript
type JarvisAudioState = 
  | 'idle'          // mic off, wake word loop running (desktop) or waiting for tap (mobile)
  | 'listening'     // actively capturing user command
  | 'processing'    // request sent to Gemini, awaiting response
  | 'speaking'      // response being read aloud (browser TTS for now)
  | 'interrupted'   // user interrupted while speaking
  | 'error'         // something went wrong
```

#### Interrupt Detection
When in `speaking` state, the SpeechRecognition instance should be active and listening for interrupt words:
- "stop"
- "wait"  
- "hold on"
- "shut up"
- "pause"

When an interrupt word is detected:
1. Immediately cancel `window.speechSynthesis.cancel()`
2. Transition state to `interrupted`
3. After 800ms transition to `listening` so user can immediately continue
4. Log the interruption as a message in the display: "— interrupted —"

#### State Transition Guards
Add a `canTransitionTo(newState)` function that validates transitions:
- From `idle` → only `listening` or `error`
- From `listening` → only `processing` or `idle` or `error`
- From `processing` → only `speaking` or `error` or `idle`
- From `speaking` → only `idle` or `interrupted` or `error`
- From `interrupted` → only `listening` or `idle`
- From `error` → only `idle`

Invalid transitions are logged as warnings but do not crash the app.

#### TTS Architecture Prep (for future Google Cloud TTS)
Create a `TTSQueue` class even though it currently uses browser TTS:

```typescript
class TTSQueue {
  private queue: string[] = []
  private isPlaying: boolean = false
  
  enqueue(text: string): void
  // splits text into sentences, adds each to queue
  // if not currently playing, starts playback
  
  private playNext(): void
  // plays the next sentence in queue
  // on completion, calls playNext() again (recursive drain)
  // when queue empty, sets isPlaying = false, transitions state to idle
  
  interrupt(): void
  // clears queue, stops current playback
  // transitions state to interrupted
  
  private splitIntoSentences(text: string): string[]
  // splits on '. ', '! ', '? ', '\n'

  // returns array of sentence strings
```

```typescript
type IntentType = 
  | 'task_creation'     // "remind me", "add task", "schedule", "don't forget", "need to"
  | 'memory_capture'    // "remember that", "save this", "note that", "keep in mind"
  | 'task_query'        // "what do I have", "what's today", "my tasks", "pending"
```

This classification is passed to Gemini as a hint in the system context:
`"The user's message has been pre-classified as: ${intentType}. Use this as a hint when deciding whether to call a tool."`

This does not replace Gemini's judgment — it's a hint that improves tool call accuracy and latency.
