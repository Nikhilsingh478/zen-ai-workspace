FIX 3 — AUTO-SCROLL TO LATEST MESSAGE
Problem: When a new long Jarvis reply triggers the window to open, the window doesn't scroll to show the latest message — it stays at top.
Two scenarios to handle:
Scenario A — Window opens with content already there:
When isOpen transitions from false to true, scroll to bottom after the opening animation completes:
typescriptuseEffect(() => {
  if (!isOpen) return

  // Wait for open animation to finish before scrolling
  const timer = setTimeout(() => {
    if (scrollRef.current) {
      scrollRef.current.scrollTo({
        top: scrollRef.current.scrollHeight,
        behavior: 'smooth'
      })
    }
  }, 400) // matches window open animation duration

  return () => clearTimeout(timer)
}, [isOpen])
Scenario B — New message arrives while window is already open (live mode):
This is the existing auto-scroll useEffect. Make sure it's correctly gated:
typescriptuseEffect(() => {
  if (!isOpen) return
  if (mode === 'history') return // never auto-scroll history

  // Small delay so DOM has rendered the new message
  const timer = setTimeout(() => {
    if (scrollRef.current) {
      scrollRef.current.scrollTo({
        top: scrollRef.current.scrollHeight,
        behavior: 'smooth'
      })
    }
  }, 100)

  return () => clearTimeout(timer)
}, [messages, isOpen, mode])
Both effects together mean:

Window opens → scrolls to bottom after animation
New message arrives → scrolls to bottom immediately
History mode → never auto-scrolls (user browses freely)
