PART 1 — MORNING BRIEFING
File: src/lib/jarvis.ts and src/routes/jarvis.tsx
What it does
When a user opens the JARVIS page for the first time each day, Jarvis automatically delivers a spoken and displayed morning briefing covering today's tasks, priorities, and a brief motivational nudge. This should feel like a real assistant greeting you, not a system notification.
Implementation in src/lib/jarvis.ts
Add a function shouldDeliverMorningBriefing():

Reads localStorage.getItem('jarvis:last-briefing-date')
Compares it to today's date in YYYY-MM-DD format
Returns true if no briefing has been delivered today, false otherwise

Add a function markBriefingDelivered():

Sets localStorage.setItem('jarvis:last-briefing-date', todayDateString)

Add a function buildMorningBriefingPrompt(tasks):

Takes today's task list as input
Returns a Gemini prompt string that instructs Gemini to generate a morning briefing
The prompt must specify:

Greet the user naturally — vary the greeting, don't always say "Good morning"
Reference the actual date and day of week
Summarize today's tasks grouped by priority — high priority tasks mentioned first by name
If no tasks today, acknowledge that and suggest it's a good day to plan
End with one honest, non-cheesy motivational line relevant to the tasks
Keep total length under 100 words — this will be spoken aloud
Tone: warm but direct, like a sharp friend, not a corporate bot

  setIsProcessing(false)

PART 2 — GEMINI SEARCH GROUNDING + DYNAMIC UI
This is the biggest feature. Read carefully.
What it does
When the user asks Jarvis something that requires current information — news, prices, weather, directions, comparisons — Jarvis uses Gemini's built-in Google Search grounding to get real-time information, then renders the results in a dynamic UI that matches the type of query. Not hardcoded cards. Not a generic list. The UI adapts to what was searched.
Step 1 — Enable Search Grounding in src/lib/gemini.ts
Add a new method generateWithSearch(messages, systemPrompt) to the GeminiAPI class:
typescriptasync generateWithSearch(
  messages: GeminiMessage[],
  systemPrompt: string
): Promise<{ text: string; sources: SearchSource[]; searchType: SearchType }> {
  
  const payload = {
    contents: messages,
    systemInstruction: { parts: [{ text: systemPrompt }] },
    tools: [{
      googleSearch: {} // enables Gemini search grounding
    }],
    generationConfig: {
      temperature: 0.7,
      topK: 40,
      topP: 0.95,
      maxOutputTokens: 8192
    }
  }
  
  const response = await this._rawRequest(payload, this.primaryKey)
  
  // Extract grounding metadata from response
  const groundingMetadata = response?.candidates?.[0]?.groundingMetadata
  const sources: SearchSource[] = groundingMetadata?.groundingChunks?.map(chunk => ({
    title: chunk.web?.title ?? '',
    url: chunk.web?.uri ?? '',
    snippet: ''
  })) ?? []
  
  const text = response?.candidates?.[0]?.content?.parts
    ?.find(p => p.text)?.text ?? ''
  
  // Classify what type of search this was
  const searchType = classifySearchType(text, messages[messages.length - 1]?.parts?.[0]?.text ?? '')
  
  return { text, sources, searchType }
}
Add these TypeScript types:
typescripttype SearchSource = {
  title: string
  url: string
  snippet: string
}

type SearchType = 
  | 'general'       // default — summary with sources
  | 'news'          // recent events, news articles  
  | 'weather'       // weather query
  | 'comparison'    // comparing two or more things
  | 'howto'         // how to do something, steps
  | 'definition'    // what is X
  | 'local'         // restaurants, places, nearby things
  | 'code'          // programming, technical questions
  | 'math'          // calculations, numbers
Add a classifySearchType(responseText, userQuery) function using keyword matching:
typescriptfunction classifySearchType(responseText: string, userQuery: string): SearchType {
  const q = userQuery.toLowerCase()
  
  if (/weather|temperature|rain|forecast|humid|°/.test(q)) return 'weather'
  if (/news|latest|today|happened|update|current/.test(q)) return 'news'
  if (/vs|versus|compare|difference|better|which/.test(q)) return 'comparison'
  if (/how to|how do|steps|guide|tutorial|setup/.test(q)) return 'howto'
  if (/what is|define|meaning|explain|who is/.test(q)) return 'definition'
  if (/near me|nearby|restaurant|hotel|place|location|where/.test(q)) return 'local'
  if (/code|error|bug|function|syntax|javascript|python|react/.test(q)) return 'code'
  if (/calculate|percent|formula|math|equation|\d+\s*[+\-*/]/.test(q)) return 'math'
  
  return 'general'
}
