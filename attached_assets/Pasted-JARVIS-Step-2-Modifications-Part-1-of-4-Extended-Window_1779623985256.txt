WHAT IS BEING BUILT
When a Jarvis reply exceeds 500 characters, a cinematic animation sequence triggers:
Step 1 — Sidebar collapses (if open)
         ↓ 150ms delay
Step 2 — Left column slides out to the left
         ↓ 150ms delay  
Step 3 — Jarvis center element scales down + moves to top
         ↓ 200ms delay
Step 4 — Extended window opens (Part 2 will build this)
When the window closes, everything reverses:
Window closes
     ↓
Jarvis element scales back + moves to center
     ↓ 150ms delay
Left column slides back in
     ↓ 150ms delay
Sidebar reopens (only if it was open before trigger)
All transitions use Framer Motion. No CSS transitions. No jitter. Everything smooth.

STEP 2 — SIDEBAR COLLAPSE COMMUNICATION
The sidebar lives in app-shell.tsx. The JARVIS page needs to be able to programmatically collapse it. The cleanest way to do this without prop drilling is a custom DOM event.
In src/components/app-shell.tsx:
Find where the sidebar open/close state is managed. Add a listener for a custom event jarvis:collapse-sidebar:
typescriptuseEffect(() => {
  const handleJarvisCollapse = (e: CustomEvent) => {
    const { collapse } = e.detail as { collapse: boolean }
    // collapse = true means close sidebar
    // collapse = false means reopen sidebar
    setSidebarOpen(!collapse) // or whatever the existing setter is called
  }

  window.addEventListener('jarvis:collapse-sidebar', handleJarvisCollapse as EventListener)
  return () => {
    window.removeEventListener('jarvis:collapse-sidebar', handleJarvisCollapse as EventListener)
  }
}, [])
Also expose current sidebar state via a custom event so JARVIS page can read it before triggering:

STEP 3 — ANIMATION TRIGGER FUNCTION
In src/routes/jarvis.tsx, add the animation orchestration function. This must be called after a long Jarvis reply is added to the display list:
typescriptconst triggerExtendedWindowSequence = useCallback(async (content: string) => {
  if (isAnimatingToExtended || extendedWindowOpen) return
  setIsAnimatingToExtended(true)

  // Read current sidebar state
  let sidebarCurrentlyOpen = false
  const handleSidebarState = (e: Event) => {
    sidebarCurrentlyOpen = (e as CustomEvent).detail.isOpen
  }
  window.addEventListener('jarvis:sidebar-state', handleSidebarState)
  // Trigger a state broadcast
  window.dispatchEvent(new CustomEvent('jarvis:request-sidebar-state'))
  window.removeEventListener('jarvis:sidebar-state', handleSidebarState)

  setSidebarWasOpenBeforeExtended(sidebarCurrentlyOpen)

  // Step 1 — Collapse sidebar if open
  if (sidebarCurrentlyOpen) {
    window.dispatchEvent(new CustomEvent('jarvis:collapse-sidebar', {
      detail: { collapse: true }
    }))
    await delay(250) // wait for sidebar animation
  }

  // Step 2 — Slide left column out
  setLeftColumnVisible(false)
  await delay(300) // wait for left column animation

  // Step 3 — Jarvis center element moves up and scales down
  // This is controlled by isAnimatingToExtended state which drives
  // the Framer Motion animate prop on the center element
  await delay(350) // wait for center element animation

  // Step 4 — Open extended window
  setExtendedWindowContent(content)
  setExtendedWindowOpen(true)
  setIsAnimatingToExtended(false)
}, [isAnimatingToExtended, extendedWindowOpen])

const closeExtendedWindowSequence = useCallback(async () => {
  // Close window first
  setExtendedWindowOpen(false)
  await delay(200)

  // Reverse Step 3 — center element returns to normal
  setIsAnimatingToExtended(false)
  await delay(300)

  // Reverse Step 2 — left column slides back in
  setLeftColumnVisible(true)
  await delay(300)

  // Reverse Step 1 — reopen sidebar if it was open before
  if (sidebarWasOpenBeforeExtended) {
    window.dispatchEvent(new CustomEvent('jarvis:collapse-sidebar', {
      detail: { collapse: false }
    }))
  }
}, [sidebarWasOpenBeforeExtended])

// Utility
const delay = (ms: number) => new Promise(resolve => setTimeout(resolve, ms))
