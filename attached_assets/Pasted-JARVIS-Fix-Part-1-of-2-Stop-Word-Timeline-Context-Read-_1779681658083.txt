FIX 2 — TIMELINE CONTEXT CAUSING ERRORS
Root cause: fetchTimelineContext() fetches all months with their full context and optional generated schedule. The generated schedule can be thousands of words — when injected into the system prompt it pushes the total prompt size over Gemini's limit causing a 400 error.
Fix in src/lib/jarvis.ts:
Update fetchTimelineContext() to NOT fetch generated_schedule — only fetch the user-written goals text which is short:
typescriptasync function fetchTimelineContext(): Promise<TimelineMonthContext[]> {
  try {
    const { data, error } = await supabase
      .from('timeline_months')
      .select('month_key, context') // remove generated_schedule entirely
      .neq('context', '')
      .order('month_key', { ascending: true })

    if (error || !data) return []

    return data.map(row => ({
      monthKey: row.month_key as string,
      context: row.context as string,
      generatedSchedule: null, // always null — don't inject schedules
    }))
  } catch {
    return []
  }
}
Also tighten the truncation in buildJarvisSystemPrompt from 400 chars to 200 chars per month and cap total months at 3 most recent:
typescriptconst timelineSection = timelineMonths && timelineMonths.length > 0
  ? `
MONTHLY GOALS (user's current focus areas):
${timelineMonths
  .slice(-3) // only last 3 months — most relevant
  .map(m =>
    `[${m.monthKey}]: ${m.context.slice(0, 200)}${m.context.length > 200 ? '...' : ''}`
  ).join('\n')}
`
  : ''
This keeps the timeline injection small — under 600 characters total — preventing prompt overflow.
