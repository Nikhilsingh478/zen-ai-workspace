We are now beginning PHASE 1 of transforming AI Matrix into a true native AI assistant platform.

IMPORTANT:
This is NOT a simple PWA packaging task.

We are NOT using PWABuilder.
We are NOT wrapping a website into a basic WebView APK.

We are transitioning AI Matrix into a proper native Android application using Capacitor as the foundation for future persistent Jarvis assistant architecture.

────────────────────────────────────
PROJECT CONTEXT
────────────────────────────────────

AI Matrix already contains:

* React + Vite architecture
* Horizon task/reminder system
* Supabase backend
* Firebase FCM notifications
* Jarvis assistant system
* wake-word functionality
* floating assistant orb
* immersive /jarvis page
* PWA functionality
* animations + futuristic UI
* service workers
* mobile responsive UI
* voice interaction systems

Desktop experience is already reasonably stable.

The major limitation now is:
mobile browser architecture itself.

Android Chrome/PWA environments aggressively:

* suspend microphone access
* throttle tabs
* kill background execution
* break continuous speech recognition
* create restart loops
* make wake-word systems unreliable

Therefore:
we are now moving away from “browser assistant architecture” and preparing the app for future native foreground-service assistant behavior.

────────────────────────────────────
PRIMARY GOAL OF THIS PHASE
────────────────────────────────────

PHASE 1 is ONLY about building the native Android foundation correctly.

DO NOT attempt to fully implement:

* always-on background Jarvis
* Porcupine wake-word engine
* foreground service persistence
* continuous microphone capture

Those come later.

For now:
focus ONLY on a professional Capacitor migration.
